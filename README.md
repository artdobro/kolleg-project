Аналітичний опис:

Навіщо Git у сучасних проєктах
Git потрібен для зручного керування змінами в коді та документах. Він дозволяє зберігати історію роботи, повертатися до попередніх версій і не боятися втратити важливі файли. У командній розробці Git дає змогу працювати паралельно, об’єднувати результати, створювати гілки для нових функцій і підтримувати порядок у проєкті. Сучасні програмні продукти неможливо уявити без Git, адже він забезпечує стабільність, контроль та передбачуваність роботи.

Ваш особистий досвід після виконання завдання
Після виконання практичного завдання стає зрозумілою реальна логіка роботи з репозиторієм: як фіксувати зміни, робити змістовні коміти, стежити за станом файлів і впевнено перемикатися між різними версіями. Виникає відчуття, що проєкт більше не “розсипається”, а всі зміни зібрані в одному місці та легко відстежуються.

Що виявилось найзручнішим / найважчим
Найбільш зручно працювати з історією змін та гілками, адже це дозволяє експериментувати і не боятися зіпсувати робочий варіант. Найскладнішими етапами зазвичай стають розуміння різниці між основними командами Git та розв’язання конфліктів злиття, коли кілька змін накладаються одна на одну. Такі ситуації потребують часу та уважності, але з досвідом стають звичними.

Як це застосувати у курсовому або колективному проєкті
У курсовому проєкті Git дає можливість впорядкувати всі робочі матеріали, зберігати проміжні результати та при потребі швидко повернутися до старої версії. У командній роботі він стає основою організації: кожен учасник може виконувати свою частину в окремій гілці, а злиття змін забезпечує узгодженість і контроль якості. Завдяки Git будь-який проєкт, навчальний чи командний, стає більш керованим, надійним і структурованим.
